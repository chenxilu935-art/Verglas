# 技术规范

## 栈与工具
- 前端框架：React
- 样式方案：Tailwind CSS
- 构建工具：建议使用 Vite（轻量、快速）
- 版本管理：Git

## 依赖与结构
- 入口：`src/main.jsx` 或 `src/index.jsx`
- 组件：按页面拆分 `src/components/` 和 `src/pages/`
- 数据：简历内容与页面文案建议放在 `src/data/` 中，方便中英文切换与可编辑性

## 设计实现要求
- 使用 Tailwind 语义类，避免大量内联样式
- 使用 CSS 变量或 Tailwind 插件管理渐变和玻璃效果
- 响应式：默认移动优先，支持 `sm/md/lg/xl` 布局
- 动画：优先使用 `transition` / `transform` / `opacity`，避免过重的 JS 计算

## 可编辑机制
- 初期实现：组件接受数据源，并给出“可编辑模式”入口
- 后续扩展：页面内直接替换文本、上传照片、保存 JSON 数据
- 先保证本地可编辑，再考虑外部存储或 CMS 集成

## 安全与稳定
- 不直接执行外部输入内容代码
- 保持组件单一职责，避免过度嵌套
- 每一步先做基础功能，再补动画和样式
