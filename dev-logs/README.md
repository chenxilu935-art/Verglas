# 开发日志说明

此目录用于记录每日开发事项、已完成项和代办项。

## 结构
- `dev-logs/YYYY-MM-DD.md` — 每天的开发记录文件
- `dev-logs/daily-log.js` — 生成当日日志文件的辅助脚本

## 使用方式
1. 进入 `个人网页` 目录。
2. 运行 `node dev-logs/daily-log.js`。
3. 打开当天生成的日志文件，补充“完成内容”和“待办事项”。

## 日志内容建议
- 已完成：今天开发的功能、页面、修复、优化等。
- 待办：下一步要做的功能、问题、测试项。

## 目标
- 保持开发过程可追踪。
- 避免重复工作。
- 让项目稳定、按阶段推进。
