# CLAUDE 工作指引

此文件用于定义项目开发过程中各项标准文档路径和具体工作说明。

## 主要文档路径
- `docs/requirements.md` — 项目需求与功能总结
- `docs/tech-specs.md` — 技术栈与实现规范
- `docs/design-guidelines.md` — 设计风格与交互规范
- `docs/execution-plan.md` — 开发执行步骤与迭代计划

## 开发日志路径
- `dev-logs/` — 每日开发记录与待办事项
- `dev-logs/README.md` — 日志使用说明
- `dev-logs/daily-log.js` — 生成当日日志文件的辅助脚本

## 当前工作说明
1. 风格方向：暗色编辑志风 + 海洋漂流主题（参考 rabenrifaie.com）。
2. 核心元素：真实海洋背景、纸质纹理明信片、电影感滚动叙事、CSS 3D 景深。
3. 明信片设计：漂流瓶概念——明信片缓缓漂来，展开个人介绍/故事/经历。
4. 字体：标题楷体（KaiTi）+ 正文 Arial，保持不变。
5. 按阶段推进，每个阶段一个可测试的交付物，完成后再进入下一阶段。
6. 每日记录完成情况与待办事项，避免后续重复工作。

## 当前执行阶段
参考 `docs/execution-plan.md`，共 6 个阶段：
1. 编辑志排版基础升级
2. 明信片纸质化重设计
3. 电影感滚动叙事
4. CSS 3D 景深效果
5. 整体打磨与响应式
6. 发布准备

## 使用方式
- 任何新需求、修改、补充方案，都先记录到 `docs/` 中
- 任何开发完成与待办事项，都写入 `dev-logs/` 对应日期文件
- 每日开发前运行 `node dev-logs/daily-log.js` 确保当日日志文件存在
- 每个阶段完成后更新 `docs/execution-plan.md` 中的勾选状态
- 若有新增标准文件，可继续补充到本 `CLAUDE.md` 中
